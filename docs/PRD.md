# MyScreen — 产品需求文档 (PRD)

## 1. 产品概述

**产品名称：** MyScreen

**一句话描述：** 一款 macOS 原生屏幕分区管理工具，可将屏幕划分为多个固定区域，将指定应用锁定在保留区域且永不被覆盖，并支持快速隐藏/显示。

**目标用户：** 需要在 Mac 上同时使用 iPhone 镜像（或其他辅助窗口）与主工作区的用户。

**技术栈：** Swift + AppKit + Accessibility API，原生 macOS 应用。

**最低系统要求：** macOS 14.0 (Sonoma) 及以上

---

## 2. 核心问题

- iPhone 镜像等辅助窗口容易被其他窗口覆盖，需要反复手动切换
- 辅助窗口经常被意外移动或调整大小，打断工作流
- 缺少一种简单的方式在"需要辅助窗口"和"全屏工作"之间快速切换

## 3. 特性列表

### P0 — 必须实现（MVP）

| 编号 | 特性 | 描述 |
|------|------|------|
| F-001 | 屏幕分区 | 将屏幕划分为 2 个区域：**工作区** 和 **保留区**。保留区位于屏幕边缘（左/右/上/下可配置），宽度/高度可按像素或百分比设定。支持在主屏幕或副屏幕上独立配置 |
| F-002 | 应用绑定 | 用户可指定一个应用（如 iPhone 镜像）绑定到保留区。绑定后该应用窗口自动移动并调整大小以适应保留区 |
| F-003 | 置顶不被覆盖 | 保留区内的窗口始终置顶显示（window level 高于普通窗口），其他窗口无法覆盖它 |
| F-004 | 工作区约束 | 其他所有应用的窗口在最大化/移动时自动被约束在工作区内，不会侵入保留区（类似 Dock 占据的空间） |
| F-005 | 快速隐藏/显示 | 全局快捷键（默认 `⌘⌥M`）一键隐藏/显示保留区。隐藏时工作区扩展为全屏，显示时工作区自动缩回 |
| F-006 | 菜单栏应用 | 以菜单栏图标（Menu Bar App）形式运行，不出现在 Dock 栏，点击图标可打开控制面板 |
| F-007 | 辅助功能权限引导 | 首次启动时引导用户授予辅助功能（Accessibility）权限，未授权时显示明确提示 |
| F-008 | 多显示器支持 | 完整支持主屏幕和副屏幕。用户可选择在任意显示器上独立启用分区，每个显示器可拥有独立的保留区配置（位置、大小、绑定应用）。支持显示器热插拔检测和自动适配 |

### P1 — 重要功能

| 编号 | 特性 | 描述 |
|------|------|------|
| F-009 | 多种分区布局 | 预设多种布局模板：左右分屏（7:3, 8:2）、上下分屏、L型布局等，用户可一键切换 |
| F-010 | 自定义分区比例 | 支持拖拽分割线调整工作区与保留区的比例 |
| F-011 | 快捷键自定义 | 允许用户自定义隐藏/显示的全局快捷键 |
| F-012 | 保留区位置切换 | 快捷键或菜单一键切换保留区位置（左↔右，上↔下） |
| F-013 | 窗口自动恢复 | 应用启动时自动恢复上次的分区布局和绑定的应用 |
| F-014 | 动画过渡 | 隐藏/显示保留区时使用平滑动画过渡，而非突然出现/消失 |

### P2 — 锦上添花

| 编号 | 特性 | 描述 |
|------|------|------|
| F-015 | 多保留区 | 支持同时设定多个保留区，分别绑定不同应用 |
| F-016 | 配置方案 | 保存/加载不同的分区配置方案（如"开发模式"、"演示模式"） |
| F-017 | 自动规则 | 根据当前运行的应用自动切换分区方案（如打开 Xcode 时自动启用开发布局） |
| F-018 | 保留区透明度 | 可设置保留区内窗口的透明度，减少视觉干扰 |
| F-019 | 拖拽绑定 | 将应用窗口拖入保留区即自动绑定 |
| F-020 | 开机自启动 | 支持设置为登录项，开机自动启动 |

## 4. 用户流程

### 4.1 首次使用

```
启动 MyScreen
    → 检测辅助功能权限
        → 未授权 → 引导用户到系统设置授权 → 等待授权
        → 已授权 → 进入主界面
    → 检测所有已连接的显示器，列出可用屏幕
    → 选择目标显示器（主屏幕 / 副屏幕）
    → 选择分区布局（默认：右侧 30% 为保留区）
    → 选择绑定应用（从运行中的应用列表选取，如 "iPhone 镜像"）
    → 完成设置，分区生效
```

### 4.2 日常使用

```
菜单栏图标常驻
    → 点击图标：显示控制面板（调整布局/更换绑定应用）
    → ⌘⌥M：隐藏/显示保留区
    → 其他窗口自动限制在工作区内
    → 保留区应用始终可见、不被遮挡
```

## 5. 技术架构概要

```
┌─────────────────────────────────────────┐
│              MyScreen App               │
├──────────┬──────────┬───────────────────┤
│ 菜单栏UI │ 设置面板  │    控制面板       │
├──────────┴──────────┴───────────────────┤
│            核心引擎 (ScreenManager)      │
├──────────┬──────────┬───────────────────┤
│ 分区计算  │ 窗口监控  │   快捷键管理      │
│ LayoutEng│ WindowMon│   HotkeyMgr      │
├──────────┴──────────┴───────────────────┤
│          系统 API 层                     │
├──────────┬──────────┬───────────────────┤
│AXUIEleme│CGDisplay │NSEvent Global     │
│Accessibi│ScreenAPI │Monitor            │
└──────────┴──────────┴───────────────────┘
```

### 关键技术点

| 模块 | 使用的 API | 作用 |
|------|-----------|------|
| 窗口控制 | `AXUIElement` (Accessibility API) | 移动/调整其他应用窗口 |
| 窗口置顶 | `AXUIElement` 设置窗口层级 / 透明屏障窗口 | 保证保留区不被覆盖 |
| 屏幕信息 | `NSScreen` / `CGDisplay` | 获取屏幕尺寸、多显示器信息 |
| 窗口监听 | `AXObserver` + `NSWorkspace` 通知 | 实时监控窗口创建/移动/调整 |
| 全局快捷键 | `CGEvent` / `NSEvent.addGlobalMonitorForEvents` | 注册全局热键 |
| 菜单栏 | `NSStatusBar` / `NSStatusItem` | 菜单栏图标和弹出面板 |
| 持久化 | `UserDefaults` / `Codable` + JSON | 保存用户配置 |

## 6. 非功能需求

| 项目 | 要求 |
|------|------|
| 性能 | 窗口监控响应延迟 < 100ms，CPU 空闲占用 < 1% |
| 内存 | 常驻内存 < 50MB |
| 兼容性 | macOS 14.0 (Sonoma) 及以上 |
| 安全 | 仅请求必要的辅助功能权限，不收集用户数据 |
| 稳定性 | 绑定的应用崩溃/退出后，MyScreen 优雅处理，不崩溃 |

---

## 7. 不做的事情（Out of Scope）

- 不支持 iOS / iPadOS
- 不提供窗口内容录制或截图功能
- 不替代系统级虚拟桌面 (Spaces)
- 不管理全屏模式 (Full Screen) 下的应用
